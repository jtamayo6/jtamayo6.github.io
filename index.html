Darcy: Your Public Transportation Assistant


Collaborators: Jorge Troncoso


One of the greatest obstacles that discourages people from using public transit is the difficulty in getting information. Bus signs are great at telling riders which bus routes serve their stop, but they fail to answer more complicated questions like, "Does bus X pass by stop Y?" Nowadays, thanks to voice recognition technology, we can extract all the information a user would want from a server and present it through a voice-user interface.

Our entry for the 2016 DIY with TI Intern Edition competition was Darcy, a dedicated voice assistant that answers questions about DART (Dallas Area Rapid Transit). Meant to be installed at bus stops and train stations, Darcy physically consists of a BeagleBone Black connected to a PCB, speaker, microphone, and Wi-Fi adapter. The heart of Darcy’s software is Amazon’s Alexa Voice Service (AVS), a cloud-based service that enables voice recognition for Internet-connected devices. Alexa is most commonly used in the Amazon Echo wireless speaker; however, the AVS API allows all developers to utilize Alexa in their own products and create their own skills (voice apps).

Darcy has several features. Below is a list of commands that you can ask her:

  • “Where is my bus (or train)?” — Darcy’s main feature is telling riders when the next bus or train will arrive. Once Darcy is told which bus stop or train station she is installed at, you can save your preferred route with Darcy so that she can give the time of the next coming bus or train.

  • “Where is the next 582 bus going southbound?” – If you have already told Darcy your stop, you can ask her about any routes that service that stop, not just the one that she has saved for you.

  • “Does my route go to stop number 67890?” – Darcy can tell you whether the route she saved for you goes to a particular stop. This can be useful if you want to check whether your route will take you to your house, a museum, a restaurant, etc.

  • “How much is a ticket?” – If you’re unfamiliar with DART, Darcy can give you basic fare information so you’re ready to pay when the bus finally arrives.

Getting the BeagleBone to run as an Amazon Alexa client was straightforward, thanks to the AlexaBeagleBone open-source project on GitHub. The real programming challenge was developing Darcy as a custom Alexa skill. Creating a custom Alexa skill consists of two parts: designing the front-end interaction model, and writing the back-end code that processes user interactions.

The interaction model describes how someone should use the skill. Any custom skill is programmed to listen for certain spoken phrases, called utterances, which correspond to the user’s intended actions, called intents. For example, the WheresMyBus intent is triggered when a user asks, “Where is the next bus?” Alexa handles the challenge of converting speech into text, but it is the responsibility of the developer to account for all the possible phrases that users can say when running their skill.

The back-end code for Alexa skills is hosted through AWS Lambda, an Amazon Web Service that eliminates the hassle of maintaining a server. When a user utters a valid phrase to Darcy, Alexa packages the specified intent and any user-provided information into a JSON object and sends it to Darcy’s lambda function for processing. Darcy’s lambda function is a Python program containing the code required to validate user-provided information, communicate with DART’s servers, and formulate Alexa’s spoken responses.

Providing Darcy with DART’s transit data was a particular challenge because DART’s real-time information is not available in the standard format that other transit agencies follow (GTFS-realtime). Instead, we developed a way to utilize DART’s mobile website. DART’s website returns information about bus and train arrivals in a simple and consistent HTML format, so sending an HTTP GET request and parsing the response is trivial using Python. And because Darcy communicates directly with DART, users can trust that her information is accurate.

To complete Darcy’s design, we created a custom circuit board and mounting structure. Darcy’s PCB consists of a pushbutton to start/stop voice recordings, as well as red and green LEDs to provide feedback to users. The mounting structure was 3D printed on a Makerbot, and it holds all the components together.

IoT Power Strip


Collaborators: CJ Park


During my summer at Texas Instruments, I worked on a side project, the IoT Power Strip, alongside a fellow intern, CJ Park. The result of our work is a regular power strip that has been hacked to have energy monitoring and voice control. Powered by the Amazon Alexa Voice Service, our smart power strip allows users to vocally turn on/off individual outlets, set timer schedules for outlets, and return power consumption information for the entire power strip.

Besides the power strip, this project consists of a BeagleBone Black with Wi-Fi, a 4-channel relay board, and a power consumption circuit. In order to individually control each power outlet and simplify the wiring, some traces were connected together onto the relay board. The power consumption circuit is based off the voltage and current sensing circuits from a TI design.

For an overview of how Amazon Alexa works and how to use BeagleBone as an Alexa client, see my other project, "Darcy: Your Public Transportation Assistant". One major caveat to Alexa is that only audio messages can be transmitted between the client and server - no other data can be appended to these packets. So sending IoT data from Alexa to the BeagleBone requires an additional protocol. While Amazon offers several powerful IoT services, we decided to use PubNub, a popular Publish/Subscribe messaging API with great documentation. For our project, the BeagleBone acts as a subscriber to a PubNub server. Inside our Alexa lambda function is code that builds messages to send to the BeagleBone via this PubNub server. So when the BeagleBone sends a voice command to Alexa, it should receive an audio message confirming the command, and a separate PubNub message containing information to turn/off outlets, etc.

Check out the GitHub repo for our project here. Special thanks to Mike Erdahl, Tier and mentor, for introducing us to the BeagleBone and Alexa!

mbed Air Drums


Collaborators: Kristen Fernandez, Pratik Gangwani, Krish Ravindranath


The mbed Air Drums was our final team project for ECE 4180: Embedded Systems Design. This electronic Air Drum Kit consists of two LSM9DS1 IMUs (inertial measurement unit) to two PVC ‘drumsticks’ that measure angular velocity and acceleration in the x- and y-axes. The Air Drum Kit also utilizes two FSRs (Force Sensitive Resistors) that are placed on the floor - one to trigger the bass drum sound, and the other to switch the hi-hat sound mode (to emulate how an actual drumset works).

A user can play different drums and cymbals: the position in the x-axis determines which drum or cymbal sound is triggered. The acceleration of the movement in the y-axis determines the volume of the sound; the faster the movement, the louder the sound. All this data is handled by the mbed microcontroller (ARM-Cortex M3), which is programmed in C++.

The MIDI protocol, a robust standard in the music industry, is used to trigger notes. The drum sounds are generated by sending MIDI messages to FL Studio, a digital audio workstation (DAW). MIDI messages can be played on any similar DAW and can also be used to control MIDI-enabled instruments and synthesizers. This means that this setup could be used to play any sounds that can be generated by the DAW and it’s accompanying plugins, not just the drum sounds, and could even be hooked up to MIDI instruments and synthesizers to make interesting sounds. Furthermore, the instrument can be used to record beats and sounds made in the DAW.

RC Car and Controls Project


Collaborators: VIP Hands-on Learning Team


This project was developed under the VIP Hands-on Learning Team (VIP HoL) at Georgia Tech. VIP HoL is an interdisplinary research team that develops experimental platforms and projects to teach students fundamental engineering concepts and design principles. Some projects are designed as labs for courses, while other projects, such as this one, are meant to be completed outside of class.

The RC Car and Controls Project is essentially an off-the-shelf toy RC car whose internal electronics have been replaced by a TI MSP430 microccontroller, an ultrasonic sensor, and a custom H-bridge circuit. Running on the MSP430 is a PID control algorithm, implemented in C, that takes in readings from the distance sensor to maintain a set distance (e.g. 50 cm) away from objects in front of it.

The most important aspect of this project is its documentation. The entire process of building this project, from the hardware hacking to the algorithm tuning, is available as a series of tutorials which students can complete based upon their interests. I myself wrote the software tutorial, which explains to program the PID controller on the MSP430, going step-by-step so that students fully understand the algorithm. By completing the full set of tutorials, students can teach themselves about hardware hacking, motor control, basic microcontroller programming in C, and other important embedded systems topics.

Check out the code for this project in the GitHub repo here.
